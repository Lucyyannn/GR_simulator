# GR_simulator 仿真系统执行流

> 以配置文件 `configs/systolic_ws_128x128_c4_simple_noc_tpuv4_half_ramulator2_ssd_fast.json`
> 为基线测例，描述 **ONNXim + Ramulator2 + FEMU-bbssd 移植版** 集成系统
> 从 `main()` 到结束的完整函数调用流。

---

## 0. 测例配置摘要

| 模块 | 类型/实现 | 频率(MHz) | 关键参数 |
|------|-----------|-----------|----------|
| NPU Core × 4 | SystolicWS (128×128) | 1000 | spad=16KB, accum=4KB, FP16 |
| DRAM | DramRamulator2 (HBM2) | 1200 | 16ch, 32B req, HBM2.yaml |
| Interconnect | SimpleInterconnect | 1000 | 16 ports/core |
| SSD (新增) | Ssd (FEMU-bbssd 移植) | 跟随DRAM | 8ch x 8lun, rd=100ns, wr=200ns, base=0x800000000, cap=1TB |
| Scheduler | SimpleScheduler | — | — |

SSD 张量安置策略：place_threshold_bytes=1MB，即大于等于 1MB 的张量（权重、KV-Cache 等）被分配到 SSD 地址空间 [0x800000000, 0x800000000+1TB)。

## 1. 启动与初始化阶段

```
main()  @ src/main.cc
|
+-- CommandLineParser::parse()            解析 --config / --models_list / --mode
|
+-- initialize_config(json)  @ src/Common.cc
|     +-- 解析 core_config[0..3]  -> CoreConfig[]
|     +-- 解析 dram_type="ramulator2"  -> DramType::RAMULATOR2
|     +-- 解析 icnt_type="simple"     -> IcntType::SIMPLE
|     +-- 解析 ssd.{enabled, address_base, capacity_bytes,
|                   nchs, luns_per_ch, pg_rd_lat, pg_wr_lat, ...}
|            -> SsdSimConfig
|
+-- OperationFactory::initialize(config)
|
+-- set_ssd_placement_policy(1MB, 0x800000000, 1TB)  @ src/Common.cc
|     +-- 写全局变量 g_ssd_threshold_bytes / g_ssd_base_addr / g_ssd_capacity_bytes
|        供 Tensor::allocate_tensor() 路由大张量用
|
+-- Simulator(config, language_mode=false)  @ src/Simulator.cc
      |
      +-- (a) 计算三域 tick 周期（皮秒 ps）
      |       _core_period = 1e6/1000 = 1000 ps  (1 ns)
      |       _icnt_period = 1e6/1000 = 1000 ps
      |       _dram_period = 1e6/1200 =  833 ps
      |       _core_time = _icnt_time = _dram_time = 0
      |
      +-- (b) DramRamulator2(config)
      |       for ch in 0..15:
      |           NDPSim::Ramulator2(ch, 16, "HBM2.yaml", ...)
      |               +-- Ramulator::Factory::create_frontend()
      |               +-- Ramulator::Factory::create_memory_system()
      |
      +-- (c) Ssd(scfg, config.dram_freq=1200)  @ src/Ssd.cc
      |       _ns_per_cycle = 1000/1200 ≈ 0.8333 ns/cycle
      |       _channels.resize(8); each channel.luns.resize(8)
      |
      +-- (d) SimpleInterconnect(config)
      |
      +-- (e) for i=0..3: Core::create(i, config) -> SystolicWS
      |
      +-- (f) Scheduler::create(...) -> SimpleScheduler

加载模型:
  for each model in models_list.json:
    Model(onnx_path, ...) -> 为每个 Tensor 调用 allocate_tensor()
         +-- should_place_in_ssd(total_bytes)?   (size >= 1MB -> true)
         +-- allocate_address_placed(size, place_ssd, ssd_base)
                 false -> DRAM 线性分配
                 true  -> SSD 区域 [0x800000000, ...) 线性分配

  simulator->register_model(model)
simulator->run_simulator()
  +-- Simulator::cycle()  进入主循环
```

## 2. 主循环：Simulator::cycle()

主循环以**最小时间推进法 (min-time step)** 驱动三个时钟域。

```
while (running()):

  set_cycle_mask()
    minimum_time = MIN3(_core_time, _dram_time, _icnt_time)
    if _core_time <= minimum: mask |= CORE_MASK; _core_time += 1000 ps
    if _dram_time <= minimum: mask |= DRAM_MASK; _dram_time +=  833 ps
    if _icnt_time <= minimum: mask |= ICNT_MASK; _icnt_time += 1000 ps
    (DRAM 周期 833 ps < core/icnt 1000 ps -> DRAM 推进更频繁)

  == CORE 阶段 (if CORE_MASK) ==
  handle_model()
    +-- _scheduler->schedule_model() -> 将就绪 Model 拆解为 Tile 序列

  for core_id in 0..3:
    pop_finished_tile() -> _scheduler->finish_tile()
    if can_issue(): issue( _scheduler->get_tile() )
    _cores[core_id]->cycle()                 <- SystolicWS::cycle()
        +-- 驱动 systolic array 流水（MAC, GEMM, vector ops）
        +-- handle_ld_inst_queue() / handle_st_inst_queue()
        +-- 生成 MemoryAccess* 推入 _request_queue
               dram_address = tensor.addr + tile_offset
               (addr < 0x800000000  -> DRAM 张量)
               (addr >= 0x800000000 -> SSD  张量)
  _core_cycles++

  == DRAM/SSD 阶段 (if DRAM_MASK) ==
  _dram->cycle()
    +-- DramRamulator2::cycle()
         for ch in 0..15:
             Ramulator2::cycle()
               +-- frontend->receive_external_requests()  (提交 request_queue 中的请求)
               |     注册 lambda callback -> 完成时推入 return_queue
               +-- memory_system->tick()    <- DRAM Controller + Bank 状态推进

  if (_ssd): _ssd->cycle()
    +-- _cycles++
        while !_pending.empty() && top.finish_cycle <= _cycles:
            access->request=false; access->dram_finish_cycle=_cycles
            _finished.push(access)

  == ICNT 阶段 (if ICNT_MASK) ==

  (A) Core -> ICNT 注入:
      for each core/port:
          if has_memory_request():
              req = top_memory_request()
              if !_icnt->is_full(port, req):
                  _icnt->push(port, get_dest_node(req), req)
                  pop_memory_request()

  (B) ICNT -> Core 响应:
      if !_icnt->is_empty(port):
          _cores[core]->push_memory_response(_icnt->top())
          _icnt->pop()

  (C) [关键] ICNT -> 存储路由（DRAM 或 SSD）:
      for mem_id in 0..15:
          core_offset = n_cores * ports_per_core
          if !_icnt->is_empty(core_offset + mem_id):
              mreq = _icnt->top(core_offset + mem_id)

              // [★修正] 注入真实仿真时间
              if _ssd && _ssd->owns_address(mreq->dram_address):
                  sim_ps = MAX(MAX(_icnt_time,_dram_time),_core_time)
                  _ssd->set_current_time_ps(sim_ps)
                  if !_ssd->is_full(mreq):
                      _ssd->push(mreq);  _icnt->pop();  _nr_to_mem++
              else if !_dram->is_full(mem_id, mreq):
                  _dram->push(mem_id, mreq);  _icnt->pop();  _nr_to_mem++

  (D) DRAM -> ICNT 响应:
      if !_dram->is_empty(mem_id):
          _icnt->push(port, dest, _dram->top());  _dram->pop()

  (E) SSD -> ICNT 响应 (固定用 channel 0 作返回路径):
      if _ssd && !_ssd->is_empty():
          sresp = _ssd->top()
          if !_icnt->is_full(core_offset+0, sresp):
              _icnt->push(core_offset+0, get_dest_node(sresp), sresp)
              _ssd->pop()

  _icnt->cycle()
```

## 3. SSD 请求完整生命周期（读取 SSD 上 16 MB 权重张量的某 32B tile）

```
时刻 T（某 ICNT 阶段），Core #2 需读地址 0x800123456:

[1] SystolicWS::handle_ld_inst_queue()
      new MemoryAccess {dram_address=0x800123456, write=false,
                        request=true, size=32, core_id=2}
      -> _request_queue

[2] ICNT阶段(A): _icnt->push(port=32+noc, dest=get_dest_node(req), req)
    get_dest_node(): req->request==true
      -> dest = n_cores*ports_per_core + Dram::get_channel_id(req)
         (地址哈希到某个 channel)

[3] SimpleInterconnect 转发（latency=1 cycle）

[4] ICNT阶段(C): mreq = _icnt->top(core_offset + ch)
      _ssd->owns_address(0x800123456) = true
      sim_ps = MAX(MAX(_icnt_time,_dram_time),_core_time)   [★修正:真实时间]
      _ssd->set_current_time_ps(sim_ps)
      _ssd->push(mreq)

[5] Ssd::push(req) @ src/Ssd.cc
      address_to_ch_lun(0x800123456, &ch, &lun)
        offset  = addr - base = 0x123456
        page_id = offset / 4096 = 291
        ch  = 291 % 8 = 3
        lun = (291/8) % 8 = 4
      stime_ns = _now_ps / 1000   (真实仿真时刻，精确到 ns)
      cmd = NAND_READ
      lat_ns = ssd_advance_status(ch=3, lun=4, NAND_READ, stime_ns):
        nand_stime = max(lun.next_lun_avail, stime_ns)
        lun.next_lun_avail = nand_stime + 100 ns
        lat = 100 ns (LUN 空闲) 或更大 (LUN 有排队)
      finish_cycle = _cycles + round(100 / 0.8333) = _cycles + 120
      _pending.push({finish_cycle, req})

[6] 接下来约 120 个 DRAM tick (≈ 100 ns):
      每次 Ssd::cycle(): _cycles++; 检查 finish_cycle <= _cycles? 否 -> 跳过

[7] 第 120 次 DRAM tick 后 _cycles == finish_cycle:
      Ssd::cycle():
        req = _pending.top().access; _pending.pop()
        req->request=false; req->dram_finish_cycle=_cycles
        _finished.push(req)

[8] ICNT阶段(E):
      sresp = _ssd->top()
      _icnt->push(core_offset+0, dest_node, sresp);  _ssd->pop()

[9] 1 个 ICNT cycle 后，ICNT阶段(B):
      _cores[2]->push_memory_response(sresp)
      Core 将数据写入 SPAD -> LD 指令完成 -> tile 继续执行
```

## 4. 结束阶段

```
Simulator::running() 返回 false:
  所有 _models 空 AND 所有 Core::running()==false
  AND _icnt 空 AND _dram 空 AND _ssd->running()==false
  AND _scheduler 空

打印统计:
  for core: core->print_stats()     各 core 计算/内存空闲周期
  _icnt->print_stats()               NoC 带宽利用率
  _dram->print_stat()                Ramulator2 finalize + BW 利用率
  _ssd->print_stat()                 读/写次数、平均/最大延迟、per-channel 分布

main(): 输出 wall-clock 仿真耗时和 TPS。
```
